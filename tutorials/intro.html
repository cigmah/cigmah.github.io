What's a _?

Comment?

A comment is something you write in your program that won't be executed. It's good practise to have comments around your code to explain something.

In Python, comments start with # (the hash character), e.g. 

# This is a comment.

Variable?

A variable is a name we use to refer to something else. What it refers to can vary, hence "variable".

In Python, we assign variables with one equals sign (=), e.g. 

x = 2
print(x)

2

Think "x refers to 2".

String?

A string is a series of 'literal' characters i.e. data which refers to characters.

In Python, we put strings in either single or double quotes, e.g. 

x = 'Hi! I am a string.'
print(x)

Hi! I am a string.

Think "x refers to <this series of characters>".

Why? Python thinks any characters you write that aren't enclosed in quotes are variables.

Function?

A function is like a verb - it's something we can tell the computer to do. Functions take arguments which are like objects or data it works on.

e.g. In English, you can say "Print my essay!" Print is a function, and essay is an argument.

In Python, functions are written like variables (no quotes) and we tell the function to perform an action using brackets which enclose the arguments, e.g. 

MY_ESSAY = "MY BRILLIANT ESSAY. By Author."
print(MY_ESSAY) # MY_ESSAY needs to refer to something though...

MY BRILLIANT ESSAY. By Author.

Or another example: len is a function which finds the length of something, like a string ("find the length of this string!") 

length = len("I have 21 characters.")
print(length)

21

                +-----------+
   Input/s  ->  | Function  |   --->   Output
                +-----------+

You can define your own functions with def ("define") e.g. 

def square(x):
    return x * x

print(square(2))

4

Package/Library/Module?

A package or library is code someone else wrote that we can use, that they've "packaged up" for us and distributed.

In Python, we can use packages by importing them, e.g. 

import requests

None

We can use the code inside a package by using a dot to "access" its properties and functions under its namespace e.g. 

requests.get("https://google.com.au")

Some libraries come with Python already - these make up the standard library. Some others you need to install via pip, which is a package manager (software that…manages packages).

Index?

An index is a number referring to a place in a string or a list, or something like similar (an "iterable").

E.g. The word "world" has five characters. The character at:

  • Index 0 is "w" (by convention, we start from 0!)
  • Index 1 is "o"
  • Index 2 is "r" etc.
world
01234

In Python, you index with square brackets after whatever you want to index, e.g. 

print("world"[0])

w

You can specify a range from the first index you specify up until the last index you specify (not including the last one) with a colon, e.g. 

print("world"[0:3])

wor

You can also do this in steps e.g. "every second index" with another colon, e.g. 

print("world"[0:5:2])

wrd

List?

A list is a bunch of things in sequential order.

Like a list of strings. Or a shopping list.

In Python, lists are demarcated by square brackets, e.g. 

LIST_OF_WORDS = ["I", "am", "a", "list", "of", "words"]
print(LIST_OF_WORDS)

['I', 'am', 'a', 'list', 'of', 'words']

For Loop?

A for loop is a way to do something "for" each element in e.g. a list (or anything "iterable").

In Python, we use for loops using the word…"for", e.g. 

LIST_OF_NUMBERS = [1, 2, 3, 4]
for number in LIST_OF_NUMBERS:
    print(number + 5)

6
7
8
9

Conditionals?

A conditional or if statement is a way of "branching out possibilities", so you do something only if something is True.

In Python, you use conditionals with the words "if", "elif" (else if) and "else", e.g. 

x = 2

if x % 2 == 0:
    print("x is even!")
else:
    print("x is odd!")

x is even!

List Comprehension?

A list comprehension is a convenient way in Python to do something to items in a list and rebuild the list again at the same time.

It's like saying "for every item in this list, do something to it and put it back in the list".

In Python, you use list comprehensions by putting a for loop inside a list e.g. 

LIST_OF_NUMBERS = [1, 2, 3, 4]
LIST_OF_ADDED_NUMBERS = [number + 5 for number in LIST_OF_NUMBERS]
print(LIST_OF_ADDED_NUMBERS)

[6, 7, 8, 9]

You can do things like filter lists with it by using if statements too. 

LIST_OF_NUMBERS = [1, 2, 3, 4]
LIST_OF_FILTERED_NUMBERS = [number for number in LIST_OF_NUMBERS if number > 2]
print(LIST_OF_FILTERED_NUMBERS)

[3, 4]

Set?

A set is like a list, but contains only unique items in any order. It's like a mathematical set.

You can make sets in Python from lists (or other "iterables") using set() e.g. 

LIST_OF_NUMBERS = [1, 1, 1, 2, 3, 3, 3, 4, 4, 4, 4]
SET_OF_NUMBERS = set(LIST_OF_NUMBERS)
print(SET_OF_NUMBERS)

{1, 2, 3, 4}

Class?

A class is a piece of data which holds properties and functions inside it. It makes it convenient to hold everything under one place. When we have something that belongs to a class, we sometimes call it an object.

Don't worry too much about making your own classes yet. It's important to know how to use them though, since they come up a lot - they're used a bit like a module, but each "object" of a class holds its own variables and state.

e.g. We can make a Counter object from the collections module to easily count things. 

from collections import Counter
LIST_OF_NUMBERS = [1, 1, 1, 2, 3, 3, 4, 4, 4, 4, 4]
COUNTS = Counter(LIST_OF_NUMBERS)
print(COUNTS.most_common())

[(4, 5), (1, 3), (3, 2), (2, 1)]

So an object is an instance of a class. Objects "belong" to a class. It's like the difference between "the" and "a".

e.g. Imagine you had a piece of paper. "The" paper in front of you is an object, but "a" paper can be written on. 

# This is a new class we are defining.
class Paper:

    def __init__(self) -> None:
        self.text = ""

    def write(self, new_text: str) -> None:
        self.text += new_text + "\n"

# We'll know make an *instance* of this class.
my_paper = Paper()

# We can access its properties.
print("This is the paper at the start.")
print(my_paper.text)

# And we can call a method on it to change its state.
my_paper.write("Here is some text.")
my_paper.write("Here is some more text.")

print("This is the paper now.")
print(my_paper.text)

This is the paper at the start.

This is the paper now.
Here is some text.
Here is some more text.

Objects have attributes (things they hold, like nouns) and methods (things they can do/can have done onto them, like verbs).

Note: this type of programming is called object-oriented programming (OOP). It lets you organise code logically and make repeated operations more abstract. It's very common in Python, but it's not the only way to organise code. There are other ways, which carry different benefits (e.g. pure functional programming like in Haskell) but which are not as well supported in Python.

Type?

A type is a way to describe what something is.

The number 4 is an integer. Integer is a type. The number 4.1= is a float (a number with a decimal point). Float is a type. The string "hello" is a string. String is a type. The list [1, 2, 3, 4] is a list of integers. List of integers is a type.

In Python, you don't need to specify the type of things, but it is good practise to. Say for example, you define your square function like this and give it a number. 

def square(x):
    return x * x
print(square(4.0))

16.0

All well and good. But what if you give it a string? 

def square(x):
    return x * x
print(square("no, not me, I'm a string"))

You get a type error, because you can't multiply a string by a string. It makes no sense.

In Python 3, you can annotate functions with types to specify what "type" of things they should take, and what "type" of things they should output. Some editors, or packages (e.g. mypy) will then check your types to make sure you haven't made silly mistakes! This is called type checking or static analysis - Python doesn't come with it by default (it is known as a dynamically-typed language, so the types of variables are only known when you run your program), but it is very useful to catch your errors. 

# We can optionally add type annotations
def square(x: float) -> float:
    return x * x

Other languages known as statically-typed languages, have much better support for this than Python.

Regular Expressions + Pandas

You might find the below useful to solve the other problems (and real life problems too.)

Regular expressions (aka "regex") are a "domain-specific language" for making text search patterns.

Imagine we had thousands of the following short stems:

Margaret Smith is a 65-year-old woman who presented to the Emergency Department with shortness of breath on a background of a 48 pack-year smoking history and recurrent ankle fractures.

John Smythe is a 28-year-old man who presented to the General Practice clinic with fever and diarrhoea on a background of recent travel to Brazil. 

STEMS = """
Margaret Smith is a 65-year-old woman who presented to the Emergency Department with shortness of breath on a background of a 48 pack-year smoking history and recurrent ankle fractures.

John Smythe is a 28-year-old man who presented to the General Practice clinic with fever and diarrhoea on a background of recent travel to Brazil.
"""

How could we extract the text into a spreadsheet of names, ages, genders, presenting location, presenting symptoms and medical background?

With regexes!

Notice that each stem is of the following form:

<FIRST NAME> <LAST NAME> is a <AGE>-year-old <GENDER> who presented to the <LOCATION> with <SYMPTOMS> on a background of <MEDICAL BACKGROUND>.

We can write a regex that matches this pattern and captures the relevant information. 

import re # This is the regex package

PATTERN = (
    r'(.+) (.+) is a (\d+)-year-old (\w+) '
    r'who presented to the (.+) with (.+) '
    r'on a background of (.+).'
)

COMPILED_PATTERN = re.compile(PATTERN)

MATCHES = COMPILED_PATTERN.findall(STEMS)
print(MATCHES)

[('Margaret', 'Smith', '65', 'woman', 'Emergency Department', 'shortness of breath', 'a 48 pack-year smoking history and recurrent ankle fractures'), ('John', 'Smythe', '28', 'man', 'General Practice clinic', 'fever and diarrhoea', 'recent travel to Brazil')]

What does this mean?

  • The pattern matches anything which fits the pattern.
  • Parentheses enclose capture groups (which get extracted).
  • A dot (.) is like a wildcard, that matches any character except a new line.
  • A plus (+) means "match the previous element one or more times"
  • A \d matches any digit.
  • A \w matches any "word" character (alphanumeric).

There's much more to regexes and they can get very complex very fast, but they are an extremely useful tool implemented in almost every programming language to process regular text.

We can then put this in a spreadsheet using pandas, Python's most popular and very widely used data science package. Spreadsheets are called data frames in Pandas. 

import pandas as pd # Convention is to import pandas as two letters

DATAFRAME = pd.DataFrame(
    MATCHES,
    columns=["first_name", "last_name", "age", "gender", "location", "symptoms", "background"]
)

DATAFRAME.as_matrix()
Margaret Smith 65 woman Emergency Department shortness of breath a 48 pack-year smoking history and recurrent ankle fractures
John Smythe 28 man General Practice clinic fever and diarrhoea recent travel to Brazil

We can also convert it to a CSV if you prefer to import it somewhere else. 

print(DATAFRAME.to_csv())

,first_name,last_name,age,gender,location,symptoms,background

0,Margaret,Smith,65,woman,Emergency Department,shortness of breath,a 48 pack-year smoking history and recurrent ankle fractures

1,John,Smythe,28,man,General Practice clinic,fever and diarrhoea,recent travel to Brazil

Data in a spreadsheet is much easier to work with than raw text. We could, for example, find the average ages of patients whose symptoms include the word 'fever', or plot the distribution of patients presenting with diarrhoea to certain locations vs others.

Building a Website and/or Web App

If you want to… Then you should learn (or we can run a tute on)…
Make a personal website/content-only webpage HTML + CSS
Make an interactive client-based webapp HTML + CSS + JavaScript (or other compile-to-JS languages)
Make a browser-based game JavaScript (or other compile-to-JS languages) + Framework
Make a full-stack application HTML + CSS + JS +/- Other Language (e.g. Python)

What this is - building all those websites you access on your computer/phone.

You don't need to program to build a website, but if you want the freedom to do anything and add any interactivity/features, you'll need to know how to program it in yourself, or you'll have to use what other people have made for you.

HTML

HTML ("hypertext markup language") isn't a programming language per se (not in the sense that e.g. Python is). It's a way to define document structure and what things mean by tagging elements. 

<head>
  <title>This is a Title</title>
</head>
<body>
  <h1>This is a heading</h1>
  <p>This is a paragraph.</p>
</body>

You can build a website entirely with HTML if you want, and HTML has some features for interactivity, but it's very limited. It's like a transparent version of Microsoft Word.

CSS

CSS ("cascading style sheets") is also not a programming language per se. It's a way to define how documents look. 

body {
    background: black;
    font-size: 14px;
    text-align: right;
}

It's like a transparent version of the formatting options of Microsoft Word, or like Microsoft Publisher.

If you want to just play with HTML and CSS with real-time updates, Scratchpad.io is a little HTML/CSS sandbox in your browser.

JavaScript (and others)

You can't make HTML and CSS actually do much - they're almost purely for presenting content.

Any actual functionality is implemented in JavaScript, or languages which transpile or compile to JavaScript. 

function square(x) {
    return x*x
}

JavaScript is the only language currently supported by the majority of browsers. Thankfully, you can write in other languages and compile it to produce JavaScript e.g. Elm (what we use predominantly for all our web apps). Here's a good list of languages which can compile to JavaScript .

With JavaScript (or languages that compile to JavaScript) you can make:

  1. Interactive websites or web apps (e.g. Spotify, Moodle, Facebook)
  2. Animations
  3. Games
  4. Anything you've seen online really.

You can also make single-page applications (which is what e.g. Reddit is nowadays…), which are sometimes entirely JavaScript with HTML snippets embedded.

JavaScript is also the pain-point of security online. People can use JavaScript to do malicious things. Since everyone uses the web, and has a browser (e.g. in your pocket!), it's a messy world out there.

Hosting

You need to make your HTML/CSS/JS files available on the WWW for people to access it. You need to host it somewhere.

Some companies let you make websites without programming and host it for you too (most blogging software).

If you've made something with custom JavaScript or other features you want to control, you can purely just host the files. GitHub and Netlify are two services which provide free file hosting for you
