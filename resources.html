Quickstart

The Quickstart is a work in progress. We'll add more as time goes on. If you have any questions in the meantime, then please don't hesitate to get in contact with us.

I have never programmed before.

Welcome to the world of programming. It's a bottomless pit, similar to medicine in many ways, but you don't have to know everything to be able to do something useful.

Programming is usually done by writing programs in programming languages. When you write a program, you give a computer instructions on what you want it to do. This gives you a lot of freedom; computers are very powerful nowadays and can do a lot, from automating tasks, to analysing data, to making interactive games, to making apps…and so on.

There are an infinitely large number of programming languages out there. A quick look on Wikipedia's List of programming languages gives you a crazily long list of everything you can imagine. Likewise, Try it Online, a web platform for trying out different languages, lists 638 languages all ready for you to code in. So what do you choose?

We'd usually recommend you start with Python, one of the most popular languages today, and a language with a lot of support for almost anything you'd want to do with a computer. Python is a beginner-friendly language, and popular among many universities as a language for beginner coders. MIT OpenCourseWare for example, which publishes many great free, public, open learning resources, has a publicly available course with lectures for Python. Python's official website also provides very thorough documentation and help on getting started with the language.

We would highly recommend you learn, not just by reading documentation and watching lectures, but by doing exercises in programming languages. If you're new to programming, it may initially be hard seeing how programming can be applied. But as you gradually become familiar with programming, it will be easier to see how programming can help.

As a starter, going through the exercises on Rosalind is a good bet. Rosalind is a bioinformatics-focused set of programming problems, so a reasonably good fit for medical students. The problems range in difficulty from easy to hard, but are a great way to start applying programming "in the real world" and exercise problem solving skills. One of the issues we often find with learning by following tutorials is that it can be hard to know what to do when there is no tutorial - so practising programming through problem-solving is a great way to exercise your ability to program de novo. For the mathematically-minded, Project Euler is a similar repository of programming problems, though with much less interaction with medicine than Rosalind.

Eventually, you should start making your own side-projects - e.g. make a small desktop app to manage your todos, or write a script that will send you an automated grocery list every weekend. We believe you'll learn the most when you start coding projects, and we're very happy to help if you need some further direction. And even if you encounter lots of errors, or think your code is verbose or ugly or repetitive, keep at it because you will get better with practise. Like medicine, you never stop learning.

There's a saying for when you start programming - that every month, you should look at your code from the month before and realise it was terrible. Because you've learned better!

I want to make games.

You will most likely first need to learn a programming language (see I have never programmed before above). There are ways of making games without programming languages per se (e.g. GameMaker which was purportedly used to make the legendary game Undertale), but you will have much more flexibility and an easier time understanding game logic if you have some general familiarity with programming.

If you would like to make games, you will most often want to use a framework on top of your chosen programming language. A framework will provide you with tools to make your life easier so you don't have to write a lot of low-level code. A lot of the details are abstracted away so you can focus on game design rather than figuring out the math behind every calculation you need to display a square on the screen.

When you develop games, you tend to need to think about what platform you want to develop for first - the tools you use to build a game in the browser, for example, are very different to the tools you use to build a game for the desktop as a native game. The type of game you make also makes a big difference - you wouldn't want to program a game with heavy 3D graphics in a language like Python since Python is notoriously slow; likewise, you probably wouldn't want to make a simple puzzle game in a low-level language like C (though many people do).

For browser-based games, you will most likely want to use JavaScript and a game framework such as Phaser (2D games) or Babylon.js (3D games). Phaser and Babylon.js are popular and have a plethora of tutorials and examples to help you along.

If you don't want to use JavaScript, then alternatives include using Elm (which is what we primarily use) with SVG or WebGL, or ClojureScript with SVG or WebGL, or using frameworks like Quil or Chocolatier. The tooling around non-JavaScript languages for web games is more immature, but offers you the benefit of not having to use JavaScript.

For desktop games, you have lots of options. Almost every programming language has some sort of framework to help you make games, including Python (Pygame) and the many Lisp dialects (the Lisp Games Wiki is a good source). If you're looking for a more batteries-included full-fledged platform, then Unity is very popular and featureful and is scriptable with C#, a Java-esque programming language from Microsoft.

I want to make apps.

In progress, but for future reference when we get to writing this:

  • Usually, programming is the easiest part of making an app (but someone has to do the programming).
  • Web apps (e.g. what you're using when you use Facebook, Netflix or Moodle) typically run with a backend hosted on a server, which receives your web requests and sends you back information when you access their service. Popular backend frameworks are Django (using Python, and what we use for our Puzzle Hunt) and Rails (using Ruby), though most language have some sort of web framework available.
  • For mobile apps - there are cross-platform frameworks like Xamarin (in C# or F#) and React Native (using JavaScript, or languages which compile to JavaScript like ClojureScript, ReasonML etc.) which let you write some non-insignificant portion of your code in one language for multiple mobile platforms (e.g. iOS, Android). Otherwise, you'll often be stuck in the Java world for Android, or the Swift world for iOS if you're writing mobile apps.
I want to do research.

In progress, but for future reference when we get to writing this:

  • Programming is not usually the hard part, and a good grounding in statistics and math is a good starter.
  • Python is ubiquitous with numpy, pandas, matplotlib, TensorFlow, and every library under the sun for science and academia.
  • Other popular languages are R, Matlab, and in certain fields, Fortran (where computational performance is important, or where there's too much legacy code). Julia is an interesting emerging option.
I want to automate some computing tasks.

In progress, but for future reference when we get to writing this:

  • Scripting is a very useful tool to automate tasks on the computer. Python is a great scripting language which can run tasks automatically like fetching or scraping web page content or repeating repetitive tasks like generating forms.
I want to make art.

In progress, but for future reference when we get to writing this:

  • Processing is probably the most eminent programming language for art, and is one way many art students are introduced to programming in the first place. You can even run Processing (or Processing-like domain-specific-languages) in the browser with libraries like p5.js without having to install anything (since everyone has a browser).
  • Almost every language has some sort of way of producing graphics. We have experience with Elm using SVG and with Julia using Luxor.jl to produce graphics and have had a pleasant experience so far. We have also used Python's matplotlib (a language for graphing) to generate simulated fundi, though we wouldn't usually recommend using it for producing arbitrary graphics.
I want to make music.

In progress, but for future reference when we get to writing this:

  • A lot of programmers are also musicians. "Live-coding" music streams can be popular, where people program music using code in front of an audience as a performance (e.g. Sam Aaron's performances). It's a really interesting experience to see and incredible to see how creative you can get with a good programming language and a text editor.
  • A good beginner-friendly platform for making music with programming is Sonic Pi which uses a Ruby-like language to generate music in an interactive environment.
  • Other languages include Overtone based on Clojure, and TidalCycles based on Haskell, which provide a high-level interface to a synthesiser.
  • For classical musicians who hav an interest in musical theory and signals, we very highly recommend the book The Haskell School of Music which is a wonderful blend of programming and music with enough beginner-friendliness for beginners of both.

Programming is a fun hobby pursuit as well.

In progress, but for future reference when we get round to writing more:

  • Code Golf! What is code golf? Solving problems with short programs ("lower is better", just like golf). Code golf communities often seem at odds with people who do programming for a job as code golfing typically leads to unmaintainable code, but it can be fun. Though, apparently large swaths of the financial industry run on K, a descendent of the APL programming language known for being a particularly good language for people who don't like typing or scrolling. Conway's Game of Life is a one-liner in APL (with only the standard library which consists of fairly generic array operations). Clearly short code can be used in the real world.
  • The International Obfuscated C Contest, which leads to very interesting submissions as far away from practicality as you can imagine.

Tools

If you're just getting started, we would recommend installing Anaconda first and having a play in Jupyter (a notebook in your browser where you can run code and see the output immediately).

  • Anaconda - a wheels-included distribution of Python which has everything you need to start using Python for data (and other!) science. Comes with Jupyter, an interactive notebook that lets you code and iterate quickly by running code and printing the output in your browser.
  • Atom, Visual Studio Code etc - extensible code editors with plugins to make your life easier. If you don't mind a learning curve, try Emacs or Vim, or combine the two with Spacemacs or Doom Emacs. Emacs also comes with org-mode, a task and note management system like Workflowy with superpowers.
  • Git - a widely used tool for version control so you don't have to track progress by e.g. renaming your documents with a date or "v2" at the front. Often associated with GitHub, an online repository to host your work and collaborate.

Languages We Use for the CIGMAH Puzzle Hunt

We're happy to answer any questions you have about our setup, or if you're interested in learning how it came to be!

  • Python - a popular general-purpose programming language. We use it with the web framework Django, which provides a bunch of tools to start building a web backend in Python.
  • Elm - a pure, functional language that compiles to JavaScript. We use it for the frontend to handle all the interaction with our Python backend, and also handle things like click events, text inputs, rendering and navigation.

Coding Practice

  • Advent of Code - A 25-day coding challenge that runs through Christmas.
  • Project Euler - A large collection of coding problems with a broad range of difficulty, often with a mathematical slant.
  • Rosalind - A bioinformatics-focused platform for learning how to code.
  • Daily Coding Problem - emailed daily coding problems.
  • HackerRank, Codeforces, Codechef, LeetCode etc. - programming competitions, often algorithm-focused.
