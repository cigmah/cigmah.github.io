Text Processing

Manipulating, extracting and creating text.

Text processing is useful for almost everyone, because almost everyone accesses text on the computer in some way. On the internet, in databases, through emails, through handouts at university, research papers, messages. All of this text is electronic data and this is very powerful. Unlike physical texts, we can automatically manipulate, analyse, extract and produce texts using computers.

Some relevant subtopics:

Regular expresssions (regex).
Regular expressions are a way of matching patterns in text. Phone-numbers (e.g. (01) 2345 6789), emails (something@something.com), names (John Smith), automated reports (something DETECTED), course and unit guides (Objectives: [...]), records (History [..] Examination [..]), and it goes on. And it's not matching literal patterns or just a glorified search - regex is very flexible. Clinicians read and write a heap of text - why not learn about how computers represent it?
Character encoding.
How can text be represented in an electronic format as bunches of 0s and 1s? What if things go wrong - have you seen weird symbols or blank white boxes? How are different languages represented? It's useful to know what goes on under the hood, and you can understand the underlying basis of a lot of text representations in software. When dealing with EMR software in a multicultural world, English alphanumeric characters are not the only thing to be aware of.

Puzzles

  • (Beginner) Combo Catcher - find a drug interaction from lists of medications.
  • (Challenge) Troubled Test - analyse pathology reports to find a malfunctioning machine giving too many false positives.
  • (Challenge) HPV Profiling - analyse HPV screening test results to find a geographical location with the highest HPV16:HPV18 ratio.

Data Science

Analysing data.

Data science is important in biomedicine - particularly because we have so much data, and we're getting more of it every day. Being able to analyse data is a central part of many advances in research - discovering patterns in data (e.g. for patients with this disease, does this drug, compared to this other drug, result in better outcomes…etc.) is central to evidence-based medicine. And it's what drives our clinical guidelines.

Given that what we learn as medical students is based on data - we think it's absolutely crucial that doctors know something about data science. How did this data come to be? Who analysed it? How? And, not only that, we think doctors could really benefit from getting their hands dirty and trying it for themselves - at least to see what sort of analyses lead to our current knowledge.

Such topics are already taught in some ways in the medical curriculum of many medical schools. We think going a step further - into the programming end - is a really useful skill. There are many complexities and "realisms" in dealing with real data, and no better way to understand than to do. Plus, being able to program means you can write your own analyses with all the freedom of the computer - you don't have to wait for someone to come up with X-package to analyse that data you've obtained.

Puzzles

  • (Challenge) Troubled Test - analyse pathology reports to find a malfunctioning machine giving too many false positives (after the text processing part, we intend for you to run some statistical tests to determine the malfunctioning machine.)
  • (Beginner) While You're Waiting - classify patients symptoms' of allergic rhinitis and count them.
  • (Beginner) Peak Flow Diary - determine the day when a patient's peak flow, standardised by weekday, was lowest.

Game Programming and Programming for Interactivity

Making games and interactive materials.

All video games are made with programs. There are different ways to make these programs, some of which don't involve programming yourself, but under the hood, that is what must happen. And being able to control things from under the hood allows you to do anything (almost).

We like making small web-based games, and we think being able to make your own games is a fun way to learn. Programming doesn't have to be all about work - it's a fun hobby! And games are an enjoyable medium of communication.

  • Explorables are learning tools making full use of the interactive capabilities of the web (and by coding, you can make your own to show anything you wish in whatever capacity you like). We also like Distill (highly interactive scientific papers about machine learning), Nicky Case (who publishes many very approachable explorables on interesting social topics) and Bret Victor (who has a number of extremely interesting projects on interaction with computers such as Dynamicland).

Puzzles

  • (Abstract) Out of Sync - manipulate parameters for a 3D cell automata ventricle model in real time.
  • (Abstract) Tarsal Tumble - play a version of the falling blocks game you know with tarsal blocks instead of the usual pieces - and build a foot out of them.
  • (Abstract) NeuroLogic - use logical reasoning to figure out locations of "lesions" on a network of "nerves" based on their result.
  • (Abstract) Base in Balance - balance a ball on a pH beam given different acid-base disturbances.
  • (Beginner) Worst Offender - win a trick-taking card game based on side-effect profiles of antipsychotics.
  • (Challenge) STI Bugs'n'Drugs - play a tile-matching game based on common STI pathogens and their medical treatment of choice.
  • (Abstract) What Dipstick? - answer questions about expected urine disptick abnormalities using an interactive dipstick.

Artificial Intelligence

Making computers "smart".

Artificial intelligence gets talked about a lot nowadays. "It's going to take our jobs", "it's not going to take our jobs" etc.

Whether or not AI does anything to our jobs, it's incredible to see what it's capable of. Computers can interpret images, reason about diagnoses, outperform humans in some cases even. They're replicable. They're fast. They have access to more information than anyone could digest in hundreds of lifetimes. And they're getting better every day. They're already used today, and chances are, something you use is powered by artificial intelligence (have you used Google?)

Artificial intelligence is a very broad area. Machine learning is one particular topic. But it doesn't have to be a nebulous idea - get your hands dirty and know what people are talking about by doing some of it yourself. Machine learning ranges from the very simple to the very complex, so we can definitely start simple. At its heart, you will find familiar concepts from statistics.

Puzzles

  • (Beginner) Fundal Flow - train your computer to recognise (simulated) fundal images to diagnose central retinal artery occlusions. It's worth noting we generated these images from scratch using code as well - no external images were used and every image is unique!
  • (Challenge) Bite Cell Scouting - train your computer to detect the locations of bite cells in (simulated) blood films containing red blood cells. We also generated these image ourselves, though they were much simpler than fundi.

Logic Programming

Building programs that can reason logically.

Logic programming is a different approach to programming than explicitly giving computers instructions and in some ways is more similar to formal logic than coding. Logic has ties with the old generation of artificial intelligence: if you're interested, check out the Fifth generation computer project, MYCIN for a medical example, and uses in modern AI with statistical artificial intelligence.

Learning logic programming might be a good approach for non-programmers to get into programming - often, you tell the computer what you want rather than how to get it. And logic and medicine have strong ties - there's lots of logic in clinical reasoning.

Puzzles

  • (Challenge) Infectious Network - determine who brought an infection to an isolated population using logic and some automation.
  • (Beginner) Reschedule All - schedule patient appointments without clashes given their availabilities during their day.
  • (Challenge) Allele Inference - use logical reasoning with Mendelian inheritance rules to determine the genotypes of a family tree.

App Development

Creating computer applications.

If you're a medical student, you've probably heard of Anki, the spaced-repetition software which a lot of students swear by. Did you know Anki is written in Python? Or, even better, that it's open source? You can view the code that Anki is written in on GitHub at https://github.com/dae/anki - and if you know Python, you'll be able to see how it all works in a real application many people benefit from today. You can even change the source code, build it on your own computer, and make Anki do exactly what you want. Change the algorithm, automatically generate cards from PowerPoint slides or PDFs, convert Anki cards to a website - one of the joys of open source software.

Being able to write programs that people can benefit from is very powerful. Every piece of software on every computer or piece of technology you've used - from Microsoft Word, to Moodle, to Youtube, to Google Maps, to EMR, to your phone app, to the university website, to internship matching software, to glucometers, to ECGs, to ultrasound equipment - has a team of programmers (people!) behind it. These pieces of software or technology didn't come out of thin air - people worked, wrote code, tested it, debugged it, made decisions and ultimately produced something that helps people. It's amazing when you think about how much our lives now depend on software people wrote - and all the more reason you should understand the background behind it.

Web Development

Creating websites and applications that run on the web.

Websites are often made with code. What you're looking at now, for instance, is a static website - it's just HTML files (though they're generated from a simpler document format, similar to Markdown). Nothing really interactive is going on.

What makes websites like Facebook or Youtube different? They're definitely not static. They host huge amounts of data, allow you to interact in real-time with other people, change the data around you, view rich media etc. They're built with servers in the back which handle all these things in other people's computers located around the world and process the information you send through your browser.

Our puzzle hunt website isn't a static website, although it's still relatively simple - it's made up out of a server written in Python (which holds and hides all the puzzles until they're released, as well as handling logins, registrations, and submissions) and a frontend written in Elm (which handles user interaction, logins etc.)

Websites range from very simple to very complex. But a lot of work went into making the web what it is today, and there's an absolute ton of complexity hidden from what you're doing on the web each day. What's a socket? An HTTP request? How does everything just work? How does data get sent from computer to computer safely, securely and reliably? We interact with the web every day - it's a good idea to learn how it all works so you can make sure you know the consequences of things you do (who has your data?) What does the web know about you?

We can also build things for the web. And not just static websites which host content - you can make apps for the web too. Apps which interact with users - apps which can host chat in real time, or provide interfaces to run complicated programs, or help diagnose illnesses or recommend treatments, or converse with users to provide a sounding board.
