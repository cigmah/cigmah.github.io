What is AORTA?

AORTA, or "an open revision tool for assessments", is a free and open-source web app we are developing ourselves. It currently has a notes and EMQ system. We hope it can be a free alternative to closed-source question banks and revision tools that cost money to use.

We're currently hosting AORTA at https://aorta.netlify.com.

Where is the code for AORTA?

Both the backend and frontend code are available from our GitHub organisation as aorticroot and aorta respectively. They are available under a GNU GPL v3.0 license.

What features does AORTA have?

If you're logged in, then it has most of the features you'd expect from online question tools, including liking, commenting, choice statistics, and adding public questions. There's a basic spaced repetition algorithm but it still needs testing.

If you're not logged in, you can view all notes and questions, but your responses aren't saved and you can't add new content.

We're happy to add features and have a few features on our plans for the next few months too, mostly involving user accuracy statistics.

What help do you need?

We've just finished the majority of the codebase, but we don't have much content yet. We need help to fill AORTA with some content so it can be a useful free alternative to proprietary question banks.

Adding content is separate to the codebase and doesn't require any coding knowledge. You can add content through a regular user account.

If you'd like to contribute to the main app direction, development and "official" notes, then please get in touch with us and we'll add you to our internal Slack group.

How do I make an account?

First, generate an account and password token to use with AORTA. You only need to provide a username you'd like to use. It is free.

You will be shown a generated password token on screen. Keep this token safe for future logins.

This registration system is currently a temporary measure; if there are issues, we may detach it and handle registrations through our Facebook page or CIGMAH group.

How do I contribute discussion to topics?

To contribute notes, mnemonics, questions or any other discussion to topics, click on the contributions tab and enter your note in the box. You will need to be logged in.

You can enter your text normally, or you can optionally format it with Markdown and a limited set of HTML tags. Please refer to the documentation for the Markdown package we are using for a full set of formatting markup available to you. We don't currently have a preview option, but that is in the works. If you have any issues with your formatting, or would like help, then contact us and we can fix it for you.

The notes that are shown initially on clicking a grid item are added by us internally. We need help with adding these. If you are interested, please get in touch with us - we'd really appreciate the help.

How do I add an EMQ?

To add an EMQ, go to the note you want to attach it to and click Add EMQ. You will need to be logged in.

You can also format your stem with Markdown.

Once you post an EMQ, it will go live immediately. It is public, so you may not want to add content that you're not happy for people to freely use.

Please note that EMQs must not contain copyrighted material. You can add images, as long as they are available under Creative Commons and you follow the attribution requirements of the license. Any material that is from copyrighted works, including past exams, books or proprietary question banks, will be removed and users may be banned.

How do I do questions?

You can do questions either from the Grid items, or from the Test tab.

The Grid items will change colour depending on if there are unknown questions (shades of red) or depending on how many questions are known if there are no unknown questions (shades of green), if you are logged in. Click on Study EMQs to do the items for a grid item - "due" items (as determined by the spaced repetition system) will always take precedence.

You can also do questions using the Test tab. This will probably only become useful when we have a sizeable collection of questions, but you can filter by specialty, year level, topic and domain.

If you are not logged in, you can do questions similarly but your responses will not be saved.

When you are on a question, click the choice you think is correct. After you have clicked a choice, you can see comments added to that question and can click on each choice to see what the creator wrote to explain why the choice was right or wrong.

Can I use AORTA on mobile?

Yes. We've specifically designed the website to be mobile friendly, though some of the features are not available (the mobile interface is primarily intended to do questions rather than to add them.)

If you're using Chrome on Android, you can also add AORTA as a Progressive Web App, which will package it is an app and pin it to your home screen. It still requires internet access though.

Will AORTA get more features?

Yes. AORTA is being actively developed by CIGMAH and we are in the process of filling in more features. If you are interested in driving the development of AORTA, please get in touch with us and we will add you to our internal Slack channel.

How is AORTA funded?

AORTA costs the CIGMAH development team $16 a month for hosting and database costs. We are currently funding it ourselves as we are very committed to keeping this tool free.

For users, AORTA doesn't cost anything and is free of ads. We aim to keep this running as long as possible, and possibly may keep AORTA an internal Monash tool depending on whether costs escalate.

For users, you have additional safety that AORTA is open-source. If we ever turn evil, you can just clone the source and host your own instances of the backend and frontend. Similarly, you can setup your own private local AORTA server for your own revision.
